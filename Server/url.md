# URL : Uniform Resource Locator

## Example: https://www.pragyan.dev/

Protocol: HyperText Transfer Protocol Secure
Here, https:// is Protocol. http is less secure than https

Domain: User Friendly Name of IP Address of Server
www.pragyan.dev is the domain.

Path: / : Home
/about : for about page
Nested Path:
/project/1
/Pragyan-Misra/node/File_Handelling

# Query Parameter: 
## Example: pragyan.dev/about?userid=1&a=2

page opening is about and we have passed some extra info that userid is 1 and a = 2.
Query Parameter starts with "?" and all spaces are converted into "+"
Example: https://www.youtube.com/results?search_query=node+js

### HTTP GET

Inorder to get some data from the server we provide a get request. When ever we open a browser and enter a url it always makes a "get" request by default.

### HTTP POST

When you send and mutate( change, add ) some data in server. Example: Sign-up, forms

### HTTP PUT

To upload something on server, like uploading our photos.

### HTTP PATCH

To change an existing entry, ex- name changing on Facebook, instagram database already has our entry but now we have to change it to change it we send a patch request

### HTTP DELETE

To delete something from server, like deleting your facebook or instagram id.

Commonly GET and POST are used.
